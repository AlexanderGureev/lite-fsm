# lite-fsm

`lite-fsm` — TypeScript-библиотека для описания конечных автоматов и управления состоянием через явные события, состояния и переходы. Базовый runtime не зависит от UI-фреймворков; дополнительные пакеты подключают React, middleware, сохранение состояния и инструменты анализа графа.

## Быстрый старт

Минимальный runtime:

```bash copy
pnpm add @lite-fsm/core
```

С reducer-ами в стиле Immer:

```bash copy
pnpm add @lite-fsm/core @lite-fsm/middleware immer
```

React-приложение:

```bash copy
pnpm add @lite-fsm/core @lite-fsm/react @lite-fsm/middleware
```

```ts showLineNumbers copy
import { MachineManager, createMachine } from "@lite-fsm/core";

const lamp = createMachine({
  config: {
    OFF: { TURN_ON: "ON", RESET: "OFF" },
    ON: { TURN_OFF: "OFF", RESET: "OFF" },
  },
  initialState: "OFF",
  initialContext: { toggleCount: 0 },
  reducer: (slice, action, { nextState }) => ({
    state: nextState,
    context: {
      toggleCount: action.type === "RESET" ? 0 : slice.context.toggleCount + 1,
    },
  }),
});

const manager = MachineManager({ lamp });

manager.transition({ type: "TURN_ON" });
manager.transition({ type: "TURN_OFF" });

manager.getState().lamp;
```

| [`@lite-fsm/core`](/packages/core) | Машины, manager, effects, actors, snapshots, types |
| [`@lite-fsm/react`](/packages/react) | React provider, hooks, hydration boundary |

## Когда использовать lite-fsm

`lite-fsm` хорошо подходит, когда поведение приложения удобнее описать явными состояниями и событиями:

- UI-компоненты с несколькими режимами и недопустимыми переходами;
- async flows с loading, success, error, cancel и retry;
- независимые подсистемы, объединенные общим `MachineManager`;
- динамические actor instances: запросы, задачи, вкладки, элементы списка;
- SSR/hydration, snapshot restore и persistence;
- визуальный анализ графа через JSON IR.

Если нужно быстро увидеть поведение вживую, откройте [playground](/playground/). Если нужно разобрать граф проекта, используйте [visualizer](/visualizer/) или [`@lite-fsm/cli`](/packages/cli).
