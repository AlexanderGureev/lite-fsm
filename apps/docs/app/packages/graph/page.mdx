# @lite-fsm/graph

`@lite-fsm/graph` строит статическое представление lite-fsm машин: JSON IR, связи между machines/managers, diagnostics, simulator input и read-only view-model для visualizer-а. Пакет не исполняет пользовательские reducer-ы и effects.

Пакет находится в alpha-версии.

## Установка

```bash copy
pnpm add -D @lite-fsm/graph
```

## Entry points

```ts copy
import {
  analyzeLiteFsmGraph,
  compileLiteFsmGraph,
  compileLiteFsmGraphProject,
  selectMachineGraph,
} from "@lite-fsm/graph";

import { createGraphSimulator } from "@lite-fsm/graph/simulator";
import { buildGraphVisualizerModel } from "@lite-fsm/graph/view-model";
```

| Entry point | Что дает |
| --- | --- |
| `@lite-fsm/graph` | Compiler, project compiler, selector, analyzer и IR-типы |
| `@lite-fsm/graph/simulator` | Headless symbolic simulation поверх graph document |
| `@lite-fsm/graph/view-model` | Projection для visualizer/workbench без React или DOM state |

## Compiler

```ts showLineNumbers copy
import { compileLiteFsmGraph } from "@lite-fsm/graph";

const result = compileLiteFsmGraph(source, {
  filename: "store/machines/toggle.ts",
  parser: "static",
});

if (result.diagnostics.length === 0) {
  console.log(result.document.machines);
}
```

Compiler сохраняет graph config transitions, reducer branches и effect emissions как read-only данные. Он не проверяет бизнес-логику через выполнение кода, поэтому подходит для visualizer-а, CI diagnostics и offline анализа.

## Project graph

```ts showLineNumbers copy
import { compileLiteFsmGraphProject } from "@lite-fsm/graph";

const result = compileLiteFsmGraphProject({
  entryFileName: "/repo/store/index.ts",
  projectRoot: "/repo",
  host,
});
```

Project compiler работает через `LiteFsmGraphProjectHost`: пакет не читает файловую систему напрямую. CLI использует этот слой для команды `lite-fsm export-graph`.

## Analyzer

```ts showLineNumbers copy
import { analyzeLiteFsmGraph } from "@lite-fsm/graph";

const analysis = analyzeLiteFsmGraph(document);
```

Analyzer возвращает diagnostics поверх готового IR. Правила alpha-версии покрывают неизвестные targets, недостижимые состояния, dead-end states, actor template shape, расхождения reducer/config, effect event acceptance и wildcard shadowing.
