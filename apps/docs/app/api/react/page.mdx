# React API

`@lite-fsm/react` помечен `"use client"`. Модуль можно импортировать из SSR/RSC, но provider и hooks должны выполняться в клиентском дереве React.

## Импорт

```ts copy
import {
  FSMContext,
  FSMContextProvider,
  FSMHydrationBoundary,
  defineMachine,
  useHydrateSnapshot,
  useManager,
  useSelector,
  useTransition,
} from "@lite-fsm/react";
```

## Компоненты

### `FSMContextProvider`

```tsx copy
<FSMContextProvider machineManager={manager}>
  <App />
</FSMContextProvider>
```

| Prop             | Назначение                                      |
| ---------------- | ----------------------------------------------- |
| `machineManager` | Экземпляр `MachineManager`                      |
| `persist?`       | Один `PersistController` или массив controllers |
| `children`       | React-поддерево                                 |

### `FSMHydrationBoundary`

```tsx copy
<FSMHydrationBoundary snapshot={snapshot} strategy="merge">
  <Page />
</FSMHydrationBoundary>
```

`FSMHydrationBoundary` даёт `useSelector()` состояние, рассчитанное из snapshot, уже во время отрисовки. После монтирования manager применяет тот же snapshot в `useLayoutEffect`.

| Prop                      | Назначение                               |
| ------------------------- | ---------------------------------------- |
| `snapshot`                | `MachineManagerSnapshot`                 |
| `strategy?`               | `"merge"` или `"replace"`                |
| `transitionAfterHydrate?` | Событие или массив событий после hydrate |
| `children`                | React-поддерево                          |

## Хуки

| Hook                                  | Что возвращает                         |
| ------------------------------------- | -------------------------------------- |
| `useSelector(selector, equalityFn?)`  | Выбранную часть manager state          |
| `useTransition()`                     | Функцию `manager.transition`           |
| `useManager()`                        | Текущий `MachineManager`               |
| `useHydrateSnapshot(snapshot, opts?)` | Применяет snapshot в `useLayoutEffect` |

```tsx showLineNumbers copy
function Counter() {
  const count = useSelector((state) => state.counter.context.count);
  const transition = useTransition();

  return <button onClick={() => transition({ type: "INC" })}>{count}</button>;
}
```

## `defineMachine`

React-пакет экспортирует `defineMachine` для сценариев без общего `MachineManager`:

```tsx copy
import { defineMachine } from "@lite-fsm/react";
```

Используйте его, когда машине не нужен общий `MachineManager` в React context. Для приложения с несколькими машинами используйте `MachineManager` + `FSMContextProvider`.

## Типы

| Тип                         | Назначение                            |
| --------------------------- | ------------------------------------- |
| `FSMContextType`            | Форма React context                   |
| `FSMContextProviderProps`   | Props провайдера                      |
| `FSMPersistLifecycle`       | Lifecycle-контракт persist-провайдера |
| `FSMHydrationBoundaryProps` | Props границы гидратации              |
| `TypedUseSelectorHook`      | Alias типизированного selector hook   |
| `TypedUseTransitionHook`    | Alias типизированного transition hook |
| `TypedUseManagerHook`       | Типизированный manager hook alias     |

Подробное руководство: [@lite-fsm/react](/packages/react).
