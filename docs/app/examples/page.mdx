# Примеры

Все примеры собраны в одном месте — папке [`playground/app/examples`](https://github.com/AlexanderGureev/lite-fsm/tree/main/playground/app/examples) репозитория. Это рабочее Next.js-приложение, которое можно запустить локально и пощупать каждый кейс вместе с DevTools.

## Запуск playground локально

```bash
git clone https://github.com/AlexanderGureev/lite-fsm.git
cd lite-fsm/playground
npm install
npm run dev
```

После запуска откройте [http://localhost:3000](http://localhost:3000) — там собраны все примеры ниже.

## С чего начать

1. Посмотрите [`examples/lamp`](https://github.com/AlexanderGureev/lite-fsm/tree/main/playground/app/examples/lamp/store) — каноничная структура: `types.ts`, `create-machine.ts`, `hooks.ts`, `index.ts` и одна машина в `machines/`.
2. Если строите SSR — изучите [`examples/ssr-demo`](https://github.com/AlexanderGureev/lite-fsm/tree/main/playground/app/examples/ssr-demo) и страницу [Сохранение состояния](/guide/hydration).
3. Для динамических подсистем (карточки, треки, чаты) — [`examples/album-download`](https://github.com/AlexanderGureev/lite-fsm/tree/main/playground/app/examples/album-download) и [`examples/likes-v2`](https://github.com/AlexanderGureev/lite-fsm/tree/main/playground/app/examples/likes-v2) и страница [Акторы](/guide/actors).
4. Если нужно сохранить состояние между сессиями или синхронизировать вкладки — посмотрите [`examples/persist`](https://github.com/AlexanderGureev/lite-fsm/tree/main/playground/app/examples/persist) и страницу [Persist](/persist).
