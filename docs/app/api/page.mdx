# API Справочник

Эта страница описывает публичный runtime API библиотеки `lite-fsm`. Подробнее по типам — в [`TYPES-CHEATSHEET.md`](https://github.com/AlexanderGureev/lite-fsm/blob/main/TYPES-CHEATSHEET.md), краткая выжимка — в [`API-CHEATSHEET.md`](https://github.com/AlexanderGureev/lite-fsm/blob/main/API-CHEATSHEET.md).

## Точки входа

| Импорт                          | Что экспортируется                                                                                                                                |
| ------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| `lite-fsm`                      | `createMachine`, `createConfig`, `createReducer`, `createEffect`, `createActorMeta`, `Machine`, `defineMachine`, `MachineManager`, `LiteFsmError` |
| `lite-fsm/middleware`           | `immerMiddleware`, `devToolsMiddleware`                                                                                                           |
| `lite-fsm/middleware/immer`     | `immerMiddleware` (отдельная точка входа)                                                                                                         |
| `lite-fsm/middleware/devTools`  | `devToolsMiddleware` (отдельная точка входа)                                                                                                      |
| `lite-fsm/react`                | `FSMContext`, `FSMContextProvider`, `FSMHydrationBoundary`, `useHydrateSnapshot`, `useManager`, `useSelector`, `useTransition`, `defineMachine`   |

`lite-fsm/react` помечен `"use client"` — модуль можно импортировать из SSR/RSC, но хуки и провайдер работают только в client tree.

## Описание автомата

Автомат описывается через `createMachine` — typed identity helper, который ничего не создаёт в runtime, а только сужает типы. Возвращает тот же объект конфигурации.

```ts showLineNumbers copy
import { createMachine } from "lite-fsm";

const counter = createMachine({
  config: {
    IDLE: { INC: "IDLE", RESET: null },
    "*": { BOOT: "IDLE" },
  },
  initialState: "IDLE",
  initialContext: { count: 0 },
  reducer: (state, action, { nextState }) => {
    state.state = nextState;
    if (action.type === "RESET") state.context.count = 0;
    if (action.type === "INC") state.context.count += 1;
  },
  effects: {
    IDLE: ({ action }) => console.log(action.type),
    "*": ({ action }) => console.log("any", action.type),
  },
});
```

| Поле                                | Назначение                                                                  |
| ----------------------------------- | --------------------------------------------------------------------------- |
| `config`                            | Граф `{ [state]: { [eventType]: target } }`. Ключ `"*"` — fallback transitions. |
| `initialState` · `initialContext`   | Стартовое `state` и `context`.                                              |
| `reducer?`                          | Чистая функция, собирающая следующее `{ state, context }`.                  |
| `effects?`                          | Sync/async side-effects по target state или `"*"`.                          |
| `hydrate?` · `dehydrate?`           | Кастомная форма snapshot для `MachineManager.hydrate()` / `dehydrate()`.    |
| `groupTag?`                         | Только для actor template — тег группы.                                     |
| `persistence?`                      | Только для actor template — `"runtime"` (по умолчанию) или `"snapshot"`.    |

### Targets

| Target                                            | Поведение                                                                |
| ------------------------------------------------- | ------------------------------------------------------------------------ |
| `"NEXT_STATE"`                                    | Переход в `"NEXT_STATE"`.                                                |
| `null`                                            | Self-transition: `state` не меняется, action всё равно дойдёт до reducer. |
| `undefined` / нет ключа                           | Action игнорируется.                                                     |
| `"__RESOLVED"` · `"__REJECTED"` · `"__CANCELLED"` | Терминальная цель для actor template — actor удаляется.                  |

Явный `config[state][type]` приоритетнее `"*"`. У actor template `__INIT` не наследует `"*"`: spawn только через явный `__INIT` transition.

### Reducer

Reducer получает три аргумента: текущий slice `{ state, context }`, action и meta-объект `{ nextState, config }`. По умолчанию ему нужно вернуть новый slice. Если подключён `immerMiddleware` — можно мутировать draft и не возвращать ничего.

```ts showLineNumbers copy
reducer: (state, action, { nextState }) => {
  state.state = nextState;
  if (action.type === "INC") {
    state.context.count += 1;
  }
};
```

Без `reducer` runtime делает default merge: `{ state: target ?? current, context: { ...context, ...action.payload } }`.

`undefined` из reducer бросает `VOID_REDUCER_ERROR`, кроме случая, когда подключено middleware с маркером `__liteFsmAllowVoidReducer` (в первую очередь — `immerMiddleware`).

### Effects

```ts showLineNumbers copy
effects: {
  SAVING: async ({ action, transition, condition, api }) => {
    await api.save(action.payload);
    await condition((e) => e.type === "ACK");
    transition({ type: "DONE" });
  },
  "*": ({ action }) => console.log(action.type),
}
```

Effect получает один объект:

| Ключ                   | Что даёт                                                                |
| ---------------------- | ----------------------------------------------------------------------- |
| `action`               | Action, прошедший middleware. Тип сужен по incoming events.             |
| `transition(action)`   | Dispatch нового события; action может включать routing `meta`.          |
| `condition(predicate)` | Promise, резолвится на ближайший action, удовлетворяющий предикату.     |
| `self`                 | (только для actor) `{ actorId, groupId, groupTag }`.                    |
| user deps              | Из `defineMachine({ dependencies })` или `manager.setDependencies(...)`. |

State-effect приоритетнее `"*"`. Wildcard срабатывает и на self-transition. Ошибки и reject из `condition` уходят в `onError` менеджера.

## Фабрики

Все фабрики — typed identity helpers. Они ничего не создают в runtime, только сужают типы.

| Factory                                      | Назначение                                                          |
| -------------------------------------------- | ------------------------------------------------------------------- |
| `createMachine(cfg)`                         | Полный machine config.                                              |
| `createConfig(cfg)`                          | Только граф переходов (для разделения).                             |
| `createReducer(fn)`                          | Reducer с фиксированным action union.                               |
| `createEffect({ effect, type?, cancelFn? })` | Оборачивает effect политикой запуска.                               |
| `createActorMeta(meta)`                      | Frozen `Readonly<ActorMeta>` для replacement / time-travel input.   |

### `createEffect`

```ts showLineNumbers copy
import { createEffect } from "lite-fsm";

const load = createEffect({
  type: "latest",
  cancelFn: ({ signal }) => () => signal.aborted,
  effect: async ({ transition }) => transition({ type: "DONE" }),
});
```

| Опция                     | Поведение                                                                            |
| ------------------------- | ------------------------------------------------------------------------------------ |
| `type: "every"` (default) | Каждый запуск может dispatch-ить независимо.                                         |
| `type: "latest"`          | Старый запуск продолжает работать, но его `transition` подавляется после нового запуска. |
| `cancelFn(deps)`          | Возвращает `cancel(): boolean`. Если вернуло `true` — `transition` подавляется.       |

В actor effect guard покрывает и `transition.unscoped/actor/group/tag`.

## `MachineManager(machines, opts?)`

Менеджер объединяет несколько автоматов, маршрутизирует события и запускает эффекты.

```ts showLineNumbers copy
import { MachineManager } from "lite-fsm";
import { immerMiddleware } from "lite-fsm/middleware/immer";
import { devToolsMiddleware } from "lite-fsm/middleware/devTools";

const manager = MachineManager(
  { counter, todos, syncActor },
  {
    middleware: [immerMiddleware, devToolsMiddleware()],
    schemaVersion: 1,
    snapshot: initialSnapshot,
    onError: console.error,
  },
);

manager.setDependencies({ api });
manager.transition({ type: "INC" });
```

### Опции

| Опция                       | Назначение                                                                                       |
| --------------------------- | ------------------------------------------------------------------------------------------------ |
| `middleware?`               | Цепочка middleware на все user actions.                                                          |
| `snapshot?`                 | Начальный snapshot, применяется со стратегией `"replace"` при создании.                          |
| `schemaVersion?`            | Версия snapshot.                                                                                 |
| `onError?`                  | Ошибки эффектов и `condition`.                                                                   |
| `onUnknownMachineKey?`      | Вызывается при unknown key в `hydrate` или начальном snapshot.                                   |
| `onSchemaVersionMismatch?`  | Вызывается при несовпадении `snapshot.schemaVersion`.                                            |
| `originId?`                 | Префикс владельца для всех созданных id. Owned id принимают форму `${originId}#${templateKey}/${counter}`. Не должен содержать `#`. |
| `generateActorId?`          | Кастомный генератор `actorId`. Получает `{ templateKey, groupTag, counter, originId, action }` и возвращает строку.                |
| `generateGroupId?`          | Кастомный генератор `groupId` для unscoped спавна. Контракт совпадает с `generateActorId`.       |

`originId` нужен для P2P, multi-tab, server↔client и других распределенных сценариев, где независимые менеджеры должны мерджить snapshot-ы без коллизий ключей. Подробнее — в разделе [Распределенный спавн](/guide/actors#распределенный-спавн).

`generateActorId` подходит для стабильных доменных id (`${originId}#player/${userId}`); рекомендуется держать хвост id в формате `.../N`, чтобы counter восстанавливался после `hydrate`. Если генератор вернет уже занятый id или не-строку, менеджер бросит `LITE_FSM_INVALID_GENERATED_ID`; пустой или содержащий `#` `originId` бросает `LITE_FSM_INVALID_OPTIONS`.

### Методы

| Метод                                | Что делает                                                                                  |
| ------------------------------------ | ------------------------------------------------------------------------------------------- |
| `getState()`                         | Текущий manager state.                                                                      |
| `transition(action)`                 | Dispatch user action. Возвращает фактически применённый `ManagerAction<P>`.                 |
| `onTransition(cb)`                   | Подписка `(prev, current, action) => void`. Возвращает unsubscribe.                         |
| `setDependencies(deps \| updater)`   | Задаёт user-deps, которые попадают в эффекты.                                               |
| `replaceReducer(enhancer)`           | Подменяет root reducer (используется middleware: Immer, DevTools restore).                  |
| `getSnapshot()`                      | Runtime snapshot `{ schemaVersion, machines }` без вызова `dehydrate` hooks.                 |
| `dehydrate(opts?)`                   | Переносимый snapshot с вызовом `dehydrate` hooks.                                           |
| `hydrate(snapshot, opts?)`           | Применяет snapshot, минуя middleware и эффекты.                                             |
| `getHydratedState(snapshot, opts?)`  | Preview результата `hydrate` без мутации manager-а.                                         |

Action идёт во все domain machines; машина без подходящего перехода остаётся без изменений (селекторы не дёргаются). Префикс `@@lite-fsm/*` зарезервирован — отправлять такие события через `transition` нельзя.

#### `transition(action)`

Принимает `{ type, payload?, meta? }`. Поле `meta` содержит routing для акторов (`actorId`, `groupId`, `groupTag`).

```ts showLineNumbers copy
manager.transition({ type: "INC" });

manager.transition({
  type: "RESOLVE",
  meta: { actorId: "request/0" },
});
```

#### `getState()`

Возвращает текущее состояние всех зарегистрированных автоматов.

```ts showLineNumbers copy
const state = manager.getState();
state.counter.context.count;
```

В DEV (`NODE_ENV !== "production"`) snapshot из `getState()` глубоко заморожен.

#### `onTransition(cb)`

Подписка на каждое успешное событие.

```ts showLineNumbers copy
const unsubscribe = manager.onTransition((prev, next, action) => {
  console.log(action.type, prev, next);
});

unsubscribe();
```

Подписка синхронная — она вызывается сразу после обновления state, до запуска эффектов.

#### `setDependencies(deps | updater)`

Передаёт пользовательские зависимости в эффекты. Принимает либо объект, либо updater-функцию.

```ts showLineNumbers copy
manager.setDependencies({
  api,
  getState: manager.getState,
});

manager.setDependencies((prev) => ({
  ...prev,
  analytics,
}));
```

#### `replaceReducer(enhancer)`

Низкоуровневый метод для middleware. Принимает enhancer `(reducer) => reducer`. В прикладном коде использовать не нужно.

#### `condition(predicate)` (через middleware API)

В среднем коде `condition` доступен внутри эффекта и middleware:

```ts showLineNumbers copy
effects: {
  PROCESSING: async ({ transition, condition }) => {
    await condition((action) => action.type === "ACK");
    transition({ type: "DONE" });
  },
}
```

`condition` возвращает promise, который резолвится на первый matching action.

## Hydration

```ts showLineNumbers copy
const snapshot = manager.dehydrate({ machines: ["counter"] });

manager.hydrate(snapshot, { strategy: "merge" });

const preview = manager.getHydratedState(snapshot, {
  strategy: "replace",
  baseState: manager.getState(),
});
```

| API                  | Hooks       | Изменяет state | Subscribers          | Effects |
| -------------------- | ----------- | -------------- | -------------------- | ------- |
| `getSnapshot()`      | —           | —              | —                    | —       |
| `dehydrate()`        | `dehydrate` | —              | —                    | —       |
| `getHydratedState()` | `hydrate`   | —              | —                    | —       |
| `hydrate()`          | `hydrate`   | да             | `@@lite-fsm/HYDRATE` | —       |

| Strategy    | Поведение                                                                                                                |
| ----------- | ------------------------------------------------------------------------------------------------------------------------ |
| `"merge"`   | Применяет только keys из snapshot, остальные машины без изменений.                                                       |
| `"replace"` | Для actor templates сбрасывает acторов, отсутствующих в snapshot. Для domain машин полную форму `replace` определяет ваш `hydrate` hook. |

Если подключён `devToolsMiddleware`, `hydrate()` отправляется в Redux DevTools как `@@lite-fsm/HYDRATE`, но по-прежнему не проходит через middleware и не запускает эффекты.

Подробное описание — в разделе [Сохранение состояния](/guide/hydration).

### Domain hooks

```ts showLineNumbers copy
const profile = createMachine({
  config: { READY: {} },
  initialState: "READY",
  initialContext: { user: null },
  dehydrate: (slice) => slice.context.user,
  hydrate: (prev, user) =>
    prev.context.user === user ? prev : { state: prev.state, context: { user } },
});
```

Возвращайте `prev`, если данные не изменились — это позволит `manager.hydrate()` не уведомлять подписчиков и не вызывать ререндер.

## Actor templates

Машина становится actor template, если в `config` есть literal `__INIT`.

```ts showLineNumbers copy
const request = createMachine({
  groupTag: "request",
  config: {
    __INIT: { START: "PENDING" },
    PENDING: { RESOLVE: "__RESOLVED", FAIL: "__REJECTED" },
  },
  initialState: "__INIT",
  initialContext: { id: "" },
});
```

| Правило            | Значение                                                       |
| ------------------ | -------------------------------------------------------------- |
| `initialState`     | Всегда `"__INIT"`.                                             |
| Spawn              | Action должен match-ить `__INIT` transition.                   |
| Public record      | `state.requests[actorId] = { state, context, meta }`           |
| Terminal targets   | `__RESOLVED` · `__REJECTED` · `__CANCELLED` удаляют actor.     |
| Domain machines    | Получают action независимо от actor routing.                   |
| Standalone runtime | Actor templates работают только внутри `MachineManager`.       |

### Routing

```ts showLineNumbers copy
manager.transition({ type: "RESOLVE", meta: { actorId: "request/0" } });
manager.transition({ type: "FAIL",    meta: { groupId: "request/0" } });
manager.transition({ type: "CANCEL",  meta: { groupTag: "request" } });
```

Приоритет: `actorId > groupId > groupTag > unscoped`. Все три принимают `string | string[]`.

Внутри actor effect доступны `self` и actor-aware dispatch:

```ts showLineNumbers copy
effects: {
  PENDING: ({ self, transition }) => {
    transition.actor(self.actorId, { type: "RESOLVE" });
    transition.group(self.groupId, { type: "PING" });
    transition.tag(self.groupTag,  { type: "BROADCAST" });
    transition.unscoped({ type: "GLOBAL_DONE" });
  },
}
```

Подробнее — в разделе [Акторы](/guide/actors).

## `defineMachine(opts?).create(cfg)` — standalone

`defineMachine` фиксирует events, deps и опции один раз и затем создаёт независимые stateful-автоматы без manager-а.

```ts showLineNumbers copy
import { defineMachine } from "lite-fsm";

const machine = defineMachine<AppEvent, Deps>({ dependencies, onError }).create(counter);

machine.transition({ type: "INC" });
machine.getState();

const off = machine.onTransition((prev, next, action) => {});
machine.addMiddleware(immerMiddleware);
```

| Метод                  | Что делает                                                                  |
| ---------------------- | --------------------------------------------------------------------------- |
| `getState()`           | Текущее `{ state, context }`.                                               |
| `transition(action)`   | Middleware → reducer → subscribers → effects.                               |
| `onTransition(cb)`     | Подписка `(prev, current, action) => void`; возвращает unsubscribe.         |
| `addMiddleware(...mw)` | Добавляет middleware в конец цепочки.                                       |

Каждый `.create(cfg)` возвращает независимую машину. `defineMachine` не поддерживает actor templates.

## `Machine(cfg)` — pure machine

Низкоуровневая фабрика без state, middleware, подписок. Полезна для тестирования reducer и effect в изоляции.

```ts showLineNumbers copy
import { Machine } from "lite-fsm";

const machine = Machine(counter);
const next = machine.transition(
  { state: "IDLE", context: { count: 0 } },
  { type: "INC" },
);
await machine.invokeEffect("IDLE", next.state, deps);
```

| Метод                               | Что делает                       |
| ----------------------------------- | -------------------------------- |
| `config`                            | Ссылка на `cfg.config`.          |
| `transition(slice, action)`         | Следующее `{ state, context }`.  |
| `invokeEffect(prev, current, deps)` | Вызывает state effect или `"*"`. |

## Wildcard

Специальный символ `"*"` в `config` означает fallback transition для любого состояния, в `effects` — fallback effect, который срабатывает на каждый dispatch (включая self-transition).

```ts showLineNumbers copy
const machine = createMachine({
  config: {
    IDLE: {
      START: "RUNNING",
    },
    RUNNING: {
      STOP: "IDLE",
    },
    "*": {
      RESET: "IDLE",
    },
  },
  initialState: "IDLE",
  initialContext: {},
  effects: {
    "*": ({ action }) => console.log("любое событие", action.type),
  },
});
```

## React API

Полное описание React-интеграции — в разделе [Использование с React](/react). Здесь — краткий справочник.

### `FSMContextProvider`

Кладёт manager в context, делая его доступным `useManager`/`useSelector`/`useTransition`.

```tsx showLineNumbers copy
import { FSMContextProvider } from "lite-fsm/react";

function App() {
  return (
    <FSMContextProvider machineManager={manager}>
      <Counter />
    </FSMContextProvider>
  );
}
```

### `useSelector(selector, equalityFn?)`

`useSyncExternalStoreWithSelector`-обёртка. Default equality — `===`.

```tsx showLineNumbers copy
const count = useSelector((state) => state.counter.context.count);
```

### `useTransition()`

Возвращает `manager.transition`.

```tsx showLineNumbers copy
const transition = useTransition();
transition({ type: "INC" });
```

### `useManager()`

Прямой доступ к `MachineManager` из контекста.

```tsx showLineNumbers copy
const manager = useManager();
const state = manager.getState();
```

### `FSMHydrationBoundary`

Применяет snapshot к React-subtree так, чтобы `useSelector()` внутри boundary видел hydrated state уже на render, а manager догонял его в layout effect.

```tsx showLineNumbers copy
import { FSMHydrationBoundary } from "lite-fsm/react";

<FSMHydrationBoundary snapshot={snapshot} strategy="merge">
  <Page />
</FSMHydrationBoundary>;
```

### `useHydrateSnapshot(snapshot, opts?)`

Применяет snapshot в layout effect без preview. Подходит для фоновой синхронизации после mount, когда временное наложение состояния во время отрисовки не нужно.

### `defineMachine` (React)

Standalone-машина в виде хука. Возвращает hook + методы standalone machine.

```tsx showLineNumbers copy
import { defineMachine } from "lite-fsm/react";

const useCounter = defineMachine<AppEvent>().create(counter);

function Counter() {
  const count = useCounter((slice) => slice.context.count);
  return <button onClick={() => useCounter.transition({ type: "INC" })}>{count}</button>;
}
```

Несколько компонентов с одним и тем же hook делят один общий state.
