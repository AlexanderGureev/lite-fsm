# lite-fsm

**lite-fsm** - это легковесная библиотека конечных автоматов (Finite State Machine) для JavaScript и TypeScript.

## Особенности

- **Простой и понятный API**: Интуитивно понятный интерфейс для определения автоматов
- **Архитектура из коробки**: Предоставляет структуру приложения (глобальное хранилище, эффекты, сервисный слой)
- **Поддержка React**: Интеграция с React через хуки и компоненты
- **Поддержка middleware**: Совместимость с подходом Redux middleware
- **Типизация**: Полная поддержка TypeScript с выводом типов
- **Компактный размер**: Минимальный объем и отсутствие тяжелых зависимостей

## Быстрый старт

### Установка

```bash
npm install lite-fsm
```

### Пример использования

```ts
import { createMachine, MachineManager } from "lite-fsm";

const playback = createMachine({
  config: {
    IDLE: {
      DO_INIT: "PAUSED",
    },
    PAUSED: {
      DO_PLAY: "PLAYING",
    },
    PLAYING: {
      DO_PAUSE: "PAUSED",
      TIME_UPDATE: null,
    },
    END: {},
  },
  initialState: "IDLE",
  initialContext: {
    currentTime: null,
    duration: null,
    remainingTime: null,
  },
  effects: {
    PLAYING: ({ services }) => services.playerService.play(),
    PAUSED: ({ services }) => services.playerService.pause(),
  },
});

const manager = MachineManager({ playback });
manager.setDependencies({
  services: {
    playerService: {
      play: () => Promise.resolve(),
      pause: () => Promise.resolve(),
    },
  },
});

manager.onTransition((prevState, nextState) => {
  console.log("[onTransition]", { prevState, nextState });
});

manager.transition({ type: "DO_INIT" });
manager.transition({ type: "DO_PLAY" });
manager.transition({
  type: "TIME_UPDATE",
  payload: { currentTime: 0, duration: 60, remainingTime: 60 },
});

const { state, context } = manager.getState().playback;
```

## Что такое Конечный Автомат?

Конечный автомат - это математическая модель вычисления, которая может находиться точно в одном из конечного числа состояний в любой момент времени. Автомат может переходить из одного состояния в другое в ответ на внешние входные данные (события или действия).

В контексте разработки ПО, конечные автоматы помогают моделировать поведение компонентов и систем, делая их более предсказуемыми и легкими для понимания.
